# 端口占用问题诊断报告

## 问题诊断结果

### 当前端口状态
- **端口 8004**: 僵尸绑定 (PID 30256 - 进程不存在)
- **端口 8005**: 僵尸绑定 (PID 26976 - 进程不存在)
- **端口 8006**: 正常运行 (PID 29792 - python.exe)

### 根本原因

**Windows TCP/IP 栈僵尸绑定** - 当进程被强制终止时，Windows 的 TCP/IP 栈可能不会立即释放端口绑定。这是 Windows 的一个已知问题。

发生场景：
1. 使用 `taskkill /F` 强制终止进程
2. Python 脚本异常退出
3. 系统资源不足时进程崩溃

### 解决方案

#### 临时解决方案（当前采用）
使用新端口 **8006** 进行开发 ✅

#### 永久解决方案

**方法1：以管理员身份重置网络栈**
```powershell
# 以管理员身份运行 PowerShell
netsh winsock reset
netsh int ipv4 reset
# 重启计算机
```

**方法2：修改动态端口范围**
```powershell
# 将动态端口范围从 1024 开始改为其他范围
netsh int ipv4 set dynamicport tcp start=49152 num=16384
```

**方法3：重启计算机**
最简单直接的方法，会清理所有僵尸绑定
