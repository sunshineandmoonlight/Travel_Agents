## 🎨 界面设计要点

### 设计原则：按钮选择为主，输入为辅

```
核心思路：
❌ 不是：让用户输入自然语言描述需求
✅ 而是：每一步提供按钮选项 + 可选补充框

优点：
• 降低用户思考成本
• 避免理解偏差
• 提高操作效率
• 移动端友好
```

## 📱 前端开发要点

### 1. 渐进式表单
- 不是一次性展示所有字段
- 根据用户输入动态展示下一步
- 每一步都可以返回修改

### 2. 富文本对话
- 支持Markdown渲染
- 系统消息和用户消息区分样式
- 支持加载状态

### 3. 卡片式选择
- 大图标，清晰描述
- 悬停效果
- 选中状态明显

### 4. 可视化对比
- 使用表格或卡片对比
- 高亮差异点
- 支持展开详情

### 5. 快捷调整
- 预设调整选项（"松一点/紧一点"）
- 自然语言输入框
- 实时预览调整效果
